巨亨 ONLINE 前端交接文件

更新日期：2026-04-09。這份文件以目前版本為準，重點包含 Banner 純圖片輪播、懸浮介面手機 / 桌機差異，以及 GitHub Actions 部署流程。

專案定位

Nuxt 3 官網型靜態網站，主打品牌展示、活動導流、排行榜、會員展示與客服入口。

內容中心

主要內容集中在 data/siteContent.ts，樣式主題集中在 assets/css/main.css。

部署方式

正式部署已改成 GitHub Actions 自動 build，不再依賴手動提交 docs/。

技術堆疊

  • Nuxt 3 + Vue 3
  • Tailwind CSS + 自訂 assets/css/main.css
  • Nuxt useState + composables
  • Google Fonts：Noto Sans TC
  • Nitro static preset

常用指令

npm install
npm run dev
npm run build
npm run preview

重要目錄

app.vue                    app 入口
assets/css/main.css        全站主題樣式、色票、輪播、modal、懸浮介面
components/                共用 UI 元件
composables/               狀態與互動邏輯
data/siteContent.ts        集中內容資料來源
layouts/default.vue        全站 layout
pages/                     各頁面路由
public/                    靜態素材
docs/                      本機 build 輸出成品
handoff/                   交接文件

互動與狀態

  • useAppState.ts：登入 modal、大廳 modal、會員狀態，並透過 localStorage 保存 jh_isLoggedIn / jh_userInfo
  • useBannerSlider.ts：Banner 自動輪播、前後切換、手機觸控滑動
  • useLeaderboardTimer.ts：排行榜更新時間

主要元件

  • AppHeader.vue：導覽、Logo、手機登入後會員資訊列
  • AppBottomNav.vue：手機底部導覽
  • AppFooter.vue：Footer 與手機 APP 下載區，桌機 / 手機結構不同
  • BannerSlider.vue：純圖片 Banner 輪播
  • FloatingPanels.vue：左右懸浮介面與手機精簡按鈕
  • LoginModal.vue、LobbyModal.vue：登入與大廳 modal

手機與桌機互動規格補充

  • 手機左側按鈕使用 fp-mobile-btn-left，已登入前往 /deposit，未登入先開 LoginModal
  • 手機右側按鈕使用 fp-mobile-btn-right，未登入開 LoginModal，已登入開 LobbyModal
  • 手機按鈕透過 lg:hidden 控制，只在手機顯示
  • 首頁使用 .content-narrow（max-width: 860px）實現 Banner 全寬、內容縮窄
  • 首頁內容主順序為：最新消息、熱門活動、熱門遊戲、排行榜快報

Footer 補充

  • 桌機版為 4 欄 grid：品牌、連結、聯絡方式、社群
  • 手機版為精簡結構：Logo、社群、APP 下載、法律聲明
  • 目前社群按鈕有 LINE、Facebook、Instagram、Telegram、X 五個
  • 社群連結目前多為 placeholder
  • 手機版 Footer 已考量 bottom nav 高度

部署注意事項

nuxt.config.ts 內設定了 app.baseURL = '/Yota_Website_Nuxt/' 與 nitro.output.publicDir = 'docs'。本機 build 仍會產生 docs/，但正式部署已改由 GitHub Actions 自動處理。

目前較偏 mock 的部分

  • 登入流程仍是前端模擬
  • 登入狀態持久化已是真實的 localStorage 實作，只有驗證邏輯仍是 mock
  • 遊戲大廳 URL 仍是 placeholder
  • 試玩 iframe 仍是示意用途
  • 排行榜與活動資料仍為靜態資料

SEO 現況與補強清單

目前已完成

  • 已採用 ssr: true + static build，搜尋引擎可讀取預渲染 HTML
  • 各主要頁面已使用 useSeoMeta
  • 首頁、活動、排行榜、儲值、教學、客服、會員頁都有基本 title / description
  • 全站已設定 lang="zh-TW"、charset、viewport

目前狀態

  • robots 仍為 noindex, nofollow
  • 網站尚未進入正式收錄階段
  • 目前 SEO 屬於「基礎結構已鋪好，但尚未正式開放索引」

上線前建議補強項目

  1. 調整 robots 為正式可收錄設定
  2. 補 robots.txt
  3. 補 sitemap.xml
  4. 補 canonical
  5. 補完整 Open Graph 與 Twitter sharing 欄位
  6. 規劃結構化資料，例如 Organization、WebSite、BreadcrumbList
  7. 確認 favicon / OG 圖實體檔已放入 public/

前端接手建議順序

  1. 等品牌文案、Banner 圖、OG 圖定稿
  2. 先補 robots.txt、sitemap.xml、canonical
  3. 再補各頁 og:image 與 social sharing 圖
  4. 最後再把 robots 開放成可收錄

補充說明

目前不建議在前期階段先把 SEO 全部做死，因為 Banner、品牌素材、最終文案、分享圖仍可能變動。現在最適合的做法是保留基礎 meta，並把正式 SEO 補強工作交接給上線前的前端階段完成。

快速原則：改內容看 data/siteContent.ts；改 Banner 看 components/BannerSlider.vue；改懸浮介面看 components/FloatingPanels.vue；改全站視覺看 assets/css/main.css。
