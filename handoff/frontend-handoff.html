FRONTEND HANDOFF

巨亨 ONLINE 前端交接文件

這份文件提供給前端接手、維護與二次開發使用，目標是讓新同事快速理解專案架構、內容來源、互動邏輯、部署方式與後續擴充方向。

專案定位

Nuxt 3 官網型靜態網站，主打品牌展示、活動導流、排行榜、會員展示與客服入口。

核心來源

內容資料集中在 data/siteContent.ts，樣式主題集中在 assets/css/main.css，build 後輸出到 docs/。

技術堆疊

  • Nuxt 3 + Vue 3
  • Tailwind CSS + 自訂 main.css
  • Nuxt useState + composables
  • Google Fonts：Noto Sans TC
  • Nitro static preset，輸出到 docs/

常用指令

npm install
npm run dev
npm run build
npm run preview

重要目錄

app.vue                    app 入口
assets/css/main.css        全站主題樣式、色票、按鈕、卡片、輪播、modal
components/                共用 UI 元件
composables/               狀態與互動邏輯
data/siteContent.ts        集中內容資料來源
layouts/default.vue        全站 layout
pages/                     各頁面路由
public/                    靜態素材
docs/                      build 後的靜態部署輸出
handoff/                   交接文件

內容資料來源

目前首頁、活動、排行榜、FAQ、會員資料都集中在 data/siteContent.ts。建議前端改內容時先從這裡下手，不要把大量靜態文案重新散落回元件。

互動與狀態

  • useAppState.ts：登入 modal、大廳 modal、會員狀態
  • useBannerSlider.ts：Banner 自動輪播、前後切換、手機觸控滑動
  • useLeaderboardTimer.ts：排行榜更新時間

主要元件

  • AppHeader.vue：頂部導覽、Logo、登入按鈕
  • AppBottomNav.vue：手機底部導覽
  • AppMarquee.vue：跑馬燈
  • BannerSlider.vue：首頁 Banner
  • LoginModal.vue：登入 / 註冊彈窗
  • LobbyModal.vue：遊戲大廳 iframe / placeholder

部署注意事項

nuxt.config.ts 內設定了 app.baseURL = '/Yota_Website_Nuxt/' 與 nitro.output.publicDir = 'docs'，代表這份專案是部署在子路徑，且正式部署檔案以 docs/ 為準。

目前較偏 mock 的部分

  • 登入流程仍是前端模擬
  • 遊戲大廳 URL 仍是 placeholder
  • 試玩 iframe 仍是示意用途
  • 排行榜與活動資料目前是靜態資料

接手建議

  • 短期先補 deposit、tutorial 的資料整合
  • 補 lint、typecheck 與共用元件抽離
  • 中期再做 API 串接與素材目錄標準化
快速原則：改內容看 data/siteContent.ts；改主題看 assets/css/main.css；改互動看 composables/ 與對應元件。
